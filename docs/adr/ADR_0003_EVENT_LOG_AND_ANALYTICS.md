# ADR-0003：学习事件日志与统计聚合分离

## 状态

已接受

## 背景

项目未来要分析刷题数量、刷题时长、单题停留、连续活跃天数、学习轨迹、平台分布、提交趋势等长期学习行为。如果只保存聚合结果，一旦统计规则变化，很难修正历史数据。

## 决策

采用“原始事件日志 + 聚合统计表”的架构。

原始事件写入：

- `activity_events`
- `problem_visits`
- `study_sessions`
- `submissions`

聚合结果写入：

- `user_daily_stats`

统计页优先读取聚合表，但聚合表必须能从原始事件重算。

## 影响

正面影响：

- 统计规则改变后可以重算。
- 学习轨迹可追溯。
- 异常数据可以诊断。
- AI 建议可以引用原始依据。

代价：

- 数据表更多。
- 需要实现重算工具。
- 需要控制事件写入频率，避免无意义膨胀。

## 活跃时间规则

停留时长和活跃时长分开：

- 停留时长：题目页打开到离开的时间。
- 活跃时长：窗口聚焦、用户非空闲时的有效时间。

默认有效活跃日：

- 当天活跃时长不少于 300 秒，或
- 当天有提交，或
- 当天有首次 AC。
